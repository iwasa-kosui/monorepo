## はじめに

本書は **「育てるTypeScriptアプリケーション」シリーズの第1冊** です。
他言語（Python / Java / Go / Elixir など）の経験を持ちながら、サーバサイドTypeScriptの本番環境レベルでの実践に不安を持つエンジニアに向けて執筆しています。

ひとつの求人応募管理アプリを題材に、降ってきた要件を素朴な指示でAIエージェントに投げると生まれる「動くけれど壊れやすい」実装を出発点として、章ごとに痛みを起点にしたリファクタを進めます。完走すると、Discriminated Union による状態モデリング、Result 型と Railway Oriented Programming、Zod による境界防御、Sensitive 型による PII 保護といった原則を、降ってきた要件に対する応答として自分の手で書けるようになります。

## 本書の特徴

- **リファクタ駆動**: 要件が次々に降ってくる中で、雑な実装が変更耐性を失っていく過程を体感し、原則を「降ってきた要件への応答」として導入します
- **エージェント時代の文脈**: 各章の冒頭で「同じ要件を素朴な指示でAIエージェントに依頼するとどう実装されがちか」を例示し、原則を持たないままエージェントに依存するリスクを言語化します
- **手を動かす教材**: 各章の開始/終了状態は別リポジトリに Git tag で固定されており、`git checkout step-NN-start` で章開始時のコードに戻れます

## 読み方

- 各章は **降ってくる要件 → 既存コードで動かす → 素朴にエージェントに依頼してみる → 何が痛いのか → どう直すか → リファクタを進める → 章末の姿** の7セクション構成です
- 章ごとに 30〜45 分程度の読了時間を想定しています
- 章の順序通りに読むことを推奨しますが、特定の原則だけ先に知りたい場合は最終章のサマリから関連章へ飛ぶこともできます
