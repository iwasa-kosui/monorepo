## 言語・ランタイム

### TypeScript 6.0 RC

[Announcing TypeScript 6.0 RC](https://devblogs.microsoft.com/typescript/announcing-typescript-6-0-rc/)

TypeScript 6.0 RCがリリースされた。JavaScriptベースの最後のリリースであり、Go言語で書き直される次期バージョン7.0（ネイティブポート）への橋渡しとして位置付けられている。

- **新機能**: Temporal API型定義、`RegExp.escape`、`Map`/`WeakMap`の`getOrInsert`/`getOrInsertComputed`メソッド、`#/`で始まるサブパスインポートのサポート
- **デフォルト値の変更**: `strict: true`、`module: esnext`、`target: es2025`がデフォルトに
- **重大な非推奨化**: `target: es5`、`baseUrl`（`paths`なしでの単独使用）、`moduleResolution: node`、AMD/UMD/SystemJSモジュール形式、`outFile`などが廃止予定
- **型推論の改善**: `this`を使用しない関数での型推論がより適切に処理されるように
- **移行ツール**: `--stableTypeOrdering`フラグで6.0と7.0間の型順序の相違を診断可能

### skill-auditor: Agent Skillの監査ツール

[作ったAgent Skill、ちゃんと動いていますか？監査用のSkill、skill-auditorを作った話](https://nyosegawa.github.io/posts/skill-auditor/)

Claude Codeで複数のAgent Skillを運用する際に発生するスキル間の注意力競合（Attention競合）問題に対応するため、ポートフォリオ全体を監査する専用スキル「skill-auditor」を開発した話。

- **Whack-a-Mole問題**: あるスキルのdescriptionを強化すると隣のスキルの精度が低下するゼロサム的競争が発生
- **Agent-as-a-Judgeアプローチ**: 正解ラベルなしでセッション文脈からルーティング正誤を判定し、構造化ルブリックとconfidenceスコアで精度を担保
- **融合アーキテクチャ**: Skill Chain型（逐次パイプライン）とSub-agent型（並列分析）を組み合わせた設計
- 32スキル・472セッション・17,851ターンの分析で、平均精度0.685、10件のパッチ提案、8件の競合ペア検出

### Error.isError(): より安全なエラーチェック

[From instanceof to Error.isError: Safer Error Checking in JavaScript](https://allthingssmitty.com/2026/02/23/from-instanceof-to-error-iserror-safer-error-checking-in-javascript/)

`instanceof Error`がクロスレルム環境（iframe、Worker、SSRなど）で失敗する問題に対し、新しい`Error.isError()`メソッドがエンジンレベルで確実にエラーオブジェクトを識別する方法を提供する。

- `instanceof`はJavaScriptの異なるグローバル環境では判定が失敗する場合がある
- `Error.isError()`はエンジンレベルの内部マークを確認し、レルム横断的に対応
- Chrome 134+、Firefox 138+、Node.js 24.3+で対応済み
- ロギングシステム、テストフレームワーク、SSR環境など境界を越えるコード領域に推奨

### Durably: TypeScript用の再開可能なジョブ実行ライブラリ

[Durably Guide](https://coji.github.io/durably/guide/)

長時間実行タスクを段階的に分割し、中断後も最後の成功ステップから再開できるTypeScriptライブラリ。SQLiteにステップ結果を保存することで進捗損失を防ぐ。

- `step.run()`と`step.progress()`によるシンプルなAPIで直感的に実装可能
- サーバー（Cron、パイプライン用）、フルスタック（リアルタイムUI付き）、SPA（SQLite WASM + OPFSでオフライン対応）の3つの実行モード

- ハートビートによる障害検知で、停止したジョブを自動的に「stale」としてマークし再実行
- RedisキューやクラウドサービスなしでSQLite + WASMで動作

### Object.groupBy(): 配列のグループ化

[Object.groupBy() - MDN](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/groupBy)

イテラブルの要素をコールバック関数の戻り値に基づいてグループ化する静的メソッド。Baseline 2024として主要ブラウザで利用可能。

```javascript
const inventory = [
  { name: "asparagus", type: "vegetables", quantity: 5 },
  { name: "bananas", type: "fruit", quantity: 0 },
];
Object.groupBy(inventory, ({ type }) => type);
// { vegetables: [...], fruit: [...] }
```

- 戻り値はnullプロトタイプオブジェクトで、各プロパティがグループの要素配列を持つ
- 文字列以外のキーでグループ化したい場合は`Map.groupBy()`を使用
- `Array.prototype.reduce()`で書いていたグループ化処理を簡潔に記述可能
