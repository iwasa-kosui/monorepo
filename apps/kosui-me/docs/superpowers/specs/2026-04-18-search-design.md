# kosui.me 検索機能 設計書

## 概要

kosui.me にサイト内検索機能を追加する。Pagefind を使い、ビルド時に生成した検索インデックスをブラウザ上で利用する。`Cmd+K` で開くモーダル型の UI で、ブログ記事とトークを全文検索できる。

## 要件

- 対象: ブログ記事（25件）+ トーク（約10件）
- マッチ対象: タイトル、説明文、本文（全文検索）
- UI: `Cmd+K` / `Ctrl+K` で開くモーダル型インクリメンタル検索
- 日本語部分一致: 必須（Pagefind CJK 分割で対応）
- インフラ: Astro 5 静的サイト、Cloudflare Pages

## 技術選定: Pagefind

Pagefind を採用する。理由:

- Astro 静的サイトとの統合が成熟している
- CJK 対応が公式サポートされている
- インデックスのチャンク分割により、コンテンツ増加時もパフォーマンスが劣化しにくい
- API モードで自前 UI を構築でき、サイトのクレイモーフィズムデザインに統合できる
- デフォルト UI は使わない（デザイン統一のため）

検討した代替案:

- ビルド時 JSON インデックス + substring マッチ: 外部依存ゼロだが、ランキングの自前実装が必要。コンテンツ増加時にインデックスサイズが問題になる
- Orama: 日本語の組み込みサポートが限定的で、部分一致の精度に懸念がある

## Pagefind 統合

### ビルドパイプライン

`package.json` の `postbuild` スクリプトで `pagefind --site dist` を実行する。Pagefind が `dist` 内の HTML を解析し、検索インデックスを `dist/pagefind/` に自動生成する。

### インデックス対象の制御

Pagefind は `data-pagefind-body` 属性が1つでもサイト内に存在すると、その属性が付いた要素だけをインデックスする。この仕組みを利用して対象を制御する。

- ブログ記事ページ（`[...slug].astro`）: `<article>` に `data-pagefind-body` を付与
- トークページ（`talks/[year]/[name].astro`）: 同様に `data-pagefind-body` を付与
- その他のページ: `data-pagefind-body` を付けないことでインデックスから除外
- `private: true` の記事: `data-pagefind-ignore="all"` で除外

### メタデータ

記事ページとトークページにそれぞれ `data-pagefind-meta="type:post"` / `data-pagefind-meta="type:talk"` を付与し、検索結果で種類バッジの出し分けに利用する。

### CJK 対応

Pagefind は CJK コンテンツを検出すると自動的に CJK 分割を有効にする。追加設定は不要。

## 検索モーダル UI

### トリガー配置

- デスクトップ: ヘッダー右側のナビピルリンク列の末尾に虫眼鏡アイコンボタンを追加
- モバイル: 固定ボトムナビに4番目のタブとして「Search」を追加（虫眼鏡アイコン + ラベル）
- `Cmd+K`（Mac）/ `Ctrl+K`（Windows/Linux）のキーボードショートカットでも開く

### モーダルの構成

- 半透明バックドロップ（`backdrop-blur-sm`）で背面をぼかす
- 中央上部にダイアログパネルを表示（`max-w-2xl`, 画面上部に寄せる）
- 最上部にテキスト入力欄（角丸、`Cmd+K` のキーバッジ表示）
- 入力欄の下に検索結果リストをスクロール表示
- `Escape` キーまたはバックドロップクリックで閉じる

### 検索結果の表示

- 各結果はタイトル + マッチした本文のスニペット（Pagefind がハイライト付きで返す）
- 結果の種類（記事 / トーク）をバッジで区別
- キーボードの上下キーで結果間を移動、`Enter` で遷移

### デザイントークン

クレイモーフィズムデザインに合わせる:

- パネル: `rounded-clay` + `shadow-clay-hover`
- ダークモード: `prefers-color-scheme: dark` で既存カラー変数に追従
- `prefers-reduced-motion: reduce` でアニメーション無効化

### コンポーネント構成

- `src/components/SearchModal/SearchModal.tsx`（React コンポーネント）
- `BaseLayout.astro` に `client:idle` で配置し、全ページで利用可能にする

## データフロー

### 初期化

モーダルが初めて開かれたタイミングで `await import('/pagefind/pagefind.js')` を動的インポートする。モーダルを開くまでインデックスは読み込まない。

### 検索フロー

1. ユーザーがテキスト入力 → デバウンス（300ms）後に `pagefind.search(query)` を呼び出し
2. Pagefind はマッチするページの参照リストを返す（軽量、本文未取得）
3. 上位 5〜10 件に対して `result.data()` を呼び出し、タイトル・スニペット・URL を取得
4. 結果を React の state に反映して表示

### パフォーマンス

- Pagefind のインデックスはチャンク分割されており、マッチしたチャンクだけをフェッチする
- 35件程度のコンテンツならインデックス全体が非常に小さく、体感ほぼ即座に結果が返る
- モーダルを閉じてもインスタンスを保持し、再度開いたときは再初期化しない

## エッジケース

- 空クエリ: 結果リストを非表示にし、初期状態のまま維持
- 結果なし: 「検索結果が見つかりませんでした」のメッセージを表示
- 開発環境: `astro dev` ではビルド成果物がないため Pagefind が動かない。モーダルは開けるが「開発環境では検索を利用できません」と表示する
- Astro View Transitions: `astro:page-load` イベントでキーボードショートカットのリスナーを再登録する

## スコープ外

- 検索結果のフィルタリング（種類別絞り込み等）
- 検索のアナリティクス（検索ログ取得等）
- オフライン対応
